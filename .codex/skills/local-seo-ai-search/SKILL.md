# Local SEO AI Search

Make the site understandable to search engines, AI systems, and buyers without stuffing or filler.

## Workflow

1. Identify business entity, domain, service area, services, and buyer intent.
2. Assign one clear intent per page.
3. Build heading structure around human search language.
4. Add useful FAQs that answer buying objections and local service questions.
5. Specify schema and metadata.
6. Define internal links between homepage, services, FAQs, contact, and service-area content.

## Output

Return:
- page intent matrix,
- title/meta rules,
- heading structure,
- service page requirements,
- FAQ plan,
- LocalBusiness schema notes,
- Service schema notes,
- image alt text direction,
- internal-link plan,
- sitemap and llms.txt notes,
- thin-content risks.

## Rules

- Do not create thin duplicate service pages.
- Do not add SEO paragraphs that no customer would read.
- Prefer specific local and service language over keyword stuffing.
- If content is only there for SEO and adds no value, flag it.
