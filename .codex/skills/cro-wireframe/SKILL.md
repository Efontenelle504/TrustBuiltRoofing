# CRO Wireframe

Design the conversion path before visual styling.

## Workflow

1. Start with the primary conversion action.
2. Build the first viewport around problem, promise, proof, and action.
3. Place trust proof before heavy explanation.
4. Convert objections into sections.
5. Add CTAs after every meaningful value build.
6. Make the mobile sequence shorter, clearer, and more action-oriented than desktop.

## Required Output

Return:
- page goal,
- first viewport structure,
- full section order,
- CTA placements and wording intent,
- form placement and fields,
- trust proof locations,
- objection-to-section map,
- mobile order,
- service-page variant rules,
- what to remove if the page feels long or cluttered.

## Critique Gate

After the wireframe, run or recommend `$hyper-critique`. Anything with unclear CTA path, filler sections, or weak proof fails.
