# Offer Architect

Build the business reason the website should convert before anyone designs sections.

## Workflow

1. Identify the target buyer, their urgent problem, and their buying anxiety.
2. Define the primary conversion action and secondary actions.
3. Create the main offer promise in plain language.
4. Map objections to proof sections.
5. Define CTA hierarchy and CTA wording rules.
6. Identify trust proof required above the fold, mid-page, and near the form.

## Output

Return:
- primary offer,
- secondary offers,
- audience and intent,
- main promise,
- urgency angle,
- trust angle,
- objections and proof needed,
- CTA hierarchy,
- form friction risks,
- sections that must exist,
- sections that should not exist.

## Bar

If the offer is generic, say so. A premium site cannot hide a weak offer behind design.
