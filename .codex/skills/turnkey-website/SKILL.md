# Turnkey Website

Use this as the master website production workflow. Do not jump straight to coding when the offer, audience, conversion path, visual direction, or launch bar is unclear.

## Gates

1. Strategy: use `$brandscape-ingestor` and `$offer-architect`.
2. Architecture: use `$cro-wireframe` and `$local-seo-ai-search`.
3. Visual direction: use `$premium-ui-director` and `$photo-shot-director`.
4. Build: use repo-native implementation patterns; use `$visual-ralph` when a reference image or live visual target matters.
5. Critique: use `$hyper-critique`; use `$ux-aesthetic-critic` for rendered UX/aesthetic detail.
6. Cleanup: use `$ai-slop-cleaner` on changed files.
7. QA: use `$launch-readiness-audit`, `$ultraqa`, `$code-review`, and `$security-review` when forms, uploads, APIs, auth, or payments are involved.
8. Growth: use `$postlaunch-cro`.

## Agent Routing

- Executive Producer / Batman: xhigh for strategy, major tradeoffs, and final launch veto.
- CRO Architect: xhigh for offer flow, CTA system, funnel, form friction, and section order.
- Visual Design Director: high for premium look, typography, spacing, hierarchy, and brand expression.
- Copy Chief: high for headlines, CTA wording, proof, objections, and service copy.
- SEO/AEO Architect: high for semantic structure, schema, FAQs, internal links, and entity clarity.
- Media Director: high for real-photo strategy and placeholder shot direction.
- Frontend Executor / Robin: medium for implementation.
- Code Cleaner: medium for simplification and anti-slop cleanup.
- QA / Visual Verifier: high for screenshots, interactions, build checks, and launch blockers.
- Repo Explorer: low/fast for file search and current-state facts.

Use expensive reasoning only at decision gates. Use medium execution for code changes.

## Required Artifacts

Before implementation, produce or infer:
- brand rules,
- offer and CTA hierarchy,
- homepage and service-page funnel,
- visual direction,
- copy direction,
- SEO/AEO plan,
- image shot list.

Before launch, verify:
- desktop and mobile screenshots,
- form behavior,
- CTA visibility,
- links,
- metadata and schema,
- accessibility basics,
- performance basics,
- no visual launch blockers,
- no obvious AI slop.

## Definition Of Done

The site is not done until `$hyper-critique` does not return `Reject` or `Major revision required`, core scores meet threshold, the build passes, and launch QA has no blockers.
