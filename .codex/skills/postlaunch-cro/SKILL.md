# Postlaunch CRO

Turn a launched site into an improvement system.

## Output

Return:
- analytics event map,
- form funnel events,
- CTA tracking plan,
- heatmap questions,
- A/B test backlog,
- copy tests,
- image replacement plan,
- SEO/AEO expansion plan,
- service-page improvement backlog,
- 30/60/90 day priorities.

## Rules

- Prioritize tests that can change revenue or lead quality.
- Do not test tiny visual preferences before offer, CTA, proof, or form friction.
- Replace placeholders with real proof as soon as possible.
- Use user behavior evidence after launch, not designer preference alone.
